## 概要

**adp-skills** は、Laiye ADP（Agentic Document Processing）が提供する、すぐに使える Skill コレクションです。VLM（視覚言語モデル）、LLM（大規模言語モデル）、エージェントの自律的な意思決定技術を深く融合し、従来のルールベースのフィールド抽出を、目標駆動型のエンドツーエンドのインテリジェントな自動化へと進化させます。

## アーキテクチャ: 点・線・面

| 階層 | 位置付け | 例 |
| --- | --- | --- |
| **点: 基本的な解析と抽出** | ドキュメントを理解し、フィールドを正確に抽出 | 海外請求書から金額、日付、サプライヤーを一度に抽出 |
| **線: エンドツーエンドのフロー** | 複数のステップを自動化ワークフローとして連結 | 請求書抽出 → 契約照合 → 金額検証 → ERP 同期 |
| **面: ヒューマンインザループによる反復学習** | 人による修正で継続学習を駆動 | 複数回の修正後、イタリア式の小数表記に適応 |

## 一般的な利用手順

### Step 1: ADP CLI のインストール

```bash
# 推奨: クロスプラットフォームの npm
npm install -g @laiye-adp/agentic-doc-parse-and-extract-cli
```

```bash
# Linux / macOS
curl -fsSL https://raw.githubusercontent.com/laiye-ai/adp-cli/main/scripts/adp-init.sh | bash
```

```powershell
# Windows PowerShell
irm https://raw.githubusercontent.com/laiye-ai/adp-cli/main/scripts/adp-init.ps1 | iex
```

### Step 2: API キーの取得

<Info>
  新規ユーザーには登録後、毎月 **100 クレジット**が無料で付与されます。
</Info>

| 地域 | ログイン URL | API ベース URL |
| --- | --- | --- |
| 中国本土 | [adp.laiye.com](https://adp.laiye.com/) | `https://adp.laiye.com/` |
| グローバル | [adp-global.laiye.com](https://adp-global.laiye.com/) | `https://adp-global.laiye.com/` |

```bash
adp config set --api-key <your-api-key>
adp config set --api-base-url https://adp.laiye.com
adp config get
```

### Step 3: APP ID の取得

| `app_type` | 種類 |
| --- | --- |
| `0` | すぐに使えるアプリ（OOTB）。`app_id` は `ootb_` で始まります |
| `1` | カスタムアプリ。ユーザーが作成します |

## 典型的なシナリオ

<AccordionGroup>
  <Accordion title="越境調達 — 海外請求書の処理">
    経理担当者の一言: 「今日届いた海外請求書を Excel にまとめて」。ADP が自動でメールをスキャンし、添付ファイルを特定、フィールドを抽出、クロスチェックを行い、サマリーレポートを生成します。
  </Accordion>

  <Accordion title="貿易物流 — 複数書類の整合性チェック">
    通関担当者の一言: 「これらの書類の荷送人・荷受人情報や貨物情報が一致しているか確認して」。ADP が請求書、パッキングリスト、提貨単などの複数書類を自動解析し、相違点レポートを出力します。
  </Accordion>
</AccordionGroup>
