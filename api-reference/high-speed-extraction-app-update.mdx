title: "修改高速版抽取应用"
description: "通过修改抽取应用 OpenAPI 更新高速版自定义抽取应用，并创建新的配置版本。"
api: "POST /open/agentic_doc_processor/{tenant_name}/v1/app-manage/update"
---

修改高速版自定义抽取应用时，复用标准的修改抽取应用接口。你需要传入高速版应用的 `app_id`、当前 `config_version` 和完整字段配置。修改成功后，接口会创建新的配置版本。

```http
POST /open/agentic_doc_processor/{tenant_name}/v1/app-manage/update
```

<Note>
  修改接口不需要传 `ai_function`。服务会按 `app_id` 定位应用；高速版应用仍然保持高速版应用类型。
</Note>

## 请求参数

| 名称 | 类型 | 是否必填 | 描述 |
| --- | --- | --- | --- |
| `tenant_name` | string | 是 | 租户名称，默认 `laiye`。 |
| `app_id` | string | 是 | 高速版自定义抽取应用 ID。 |
| `app_name` | string | 否 | 应用名称。不传或传空字符串时不更新应用名称。 |
| `config_version` | string | 是 | 当前配置版本。修改后会另存为新版本。 |
| `app_label` | string[] | 否 | 应用标签列表，最多 5 个。 |
| `extract_fields` | object[] | 是 | 抽取字段配置列表，最多 20 个字段。修改时需要传入完整字段配置。 |
| `parse_mode` | string | 是 | 解析模式。高速版建议传 `fast_vlm`。 |
| `enable_long_doc` | boolean | 否 | 是否开启长文档支持，默认 `false`。 |
| `long_doc_config` | object[] | 否 | 长文档配置，最多 5 组，仅 `enable_long_doc=true` 时使用。 |

## 字段配置

### `extract_fields[]`

| 名称 | 类型 | 是否必填 | 描述 |
| --- | --- | --- | --- |
| `field_name` | string | 是 | 字段名称。 |
| `field_type` | string | 是 | 字段类型，支持 `string`、`date`、`table`。 |
| `field_prompt` | string | 否 | 字段抽取提示词。 |
| `sub_fields` | object[] | 条件必填 | 表格子字段。`field_type=table` 时必须至少提供 1 个有效子字段。 |

### `sub_fields[]`

| 名称 | 类型 | 是否必填 | 描述 |
| --- | --- | --- | --- |
| `field_name` | string | 是 | 子字段名称。 |
| `field_type` | string | 是 | 子字段类型，支持 `string`、`date`。 |
| `field_prompt` | string | 否 | 子字段抽取提示词。 |

  --header 'Content-Type: application/json' \
  --header 'X-API-KEY: {api_key}' \
  --data '{

    "app_label": ["invoice", "high-speed"],

        "field_type": "string",

## 业务规则

1. 高速版自定义抽取应用没有独立的修改路由。
2. 修改接口按 `app_id` 定位应用；请求体不需要传 `ai_function`。
3. 修改字段配置时需要传入完整的 `extract_fields`。
4. 修改成功后会创建新的配置版本，并返回新的 `config_version`。
5. `field_type=table` 的字段必须包含至少一个有效 `sub_fields`；表格子字段不能再嵌套表格。
