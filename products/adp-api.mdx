## OpenAPI 使用指南

### 第一步: 获取API Key与APP ID

`API Key` 是调用整套 ADP OpenAPI 的**全局身份凭证**，用于校验调用方企业 / 账号访问权限，所有接口发起请求时都必须携带该凭证鉴权，无合法 API Key 的请求会被网关直接拦截。

`APP ID` 是 ADP 内单个应用的**唯一标识**，用于区分你要操作、调用的目标应用，在文档抽取、工作流执行、应用配置类接口中作为核心业务入参，确定能力归属。

#### 如何获取API Key?

你需要前往 [ADP 平台](https://adp.laiye.com/) 获取 API Key。点击个人头像后，进入 **我的 API** 查看令牌。或者点击需要调用的应用卡片右上角的"..."

## 认证方式

文档抽取、文档解析、应用管理和积分查询接口使用 `X-API-KEY` 请求头。

工作流接口使用 `X-API-Key` 请求头。

## 接口参考

你可以在左侧导航的 **ADP API** 下查看所有接口。每个接口页面包含请求参数、响应结构和调用示例。也可以在顶部搜索栏查询需要调用的接口

## 文件传入方式

ADP支持两种文件传入方式，`file_url` 和 `file_base64` 二选一即可

- `file_url`：线上文件 URL。仅支持 HTTP 开头的地址。
- `file_base64`：本地文件的 Base64 内容。

ADP 为用户免费提供 [获取文件URL接口](https://adp-doc.laiye.com/api-reference/adp-api/%E5%B7%A5%E4%BD%9C%E6%B5%81%E6%96%87%E4%BB%B6/%E6%96%87%E4%BB%B6%E4%B8%8A%E4%BC%A0) ，可上传本地文件至ADP服务器，同时我们开放已上传文件删除接口，对于有安全性较高的文档，可以在处理任务完成后调用删除接口删除文件。
